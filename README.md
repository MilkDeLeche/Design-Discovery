## The problem

You're vibe coding. You point at a card grid and say *"make these hoverable."*

The AI either guesses wrong, or asks you to *"describe the animation you envision, including start and end states."* вҖ” and now **you** are doing the design thinking you opened the AI to avoid.

You don't always know what you want. You just know the current thing isn't it.

## The fix

**Design Discovery** is a **discoverer**, not a clarifier.

A clarifier pins down what you already had in mind. A discoverer expands what you thought was possible вҖ” *then* converges and builds it.

> You react to options far more creatively than you generate from a blank page. This skill supplies the options.

It reads your **real code first** вҖ” tokens, neighbors, conventions вҖ” then holds up a few concrete directions. Always one bolder than you asked for. You pick by *seeing*, not spec-writing.

## The loop

```mermaid
flowchart LR
  A[Locate] --> B[Read code]
  B --> C[Gauge openness]
  C --> D[Propose directions]
  D --> E[Converge]
  E --> F[Build]
  F --> G[Verify]
  G --> H{Keep or switch?}
  H -->|Switch| F
  H -->|Keep| I[Push further]
  I --> J[PROGRESS.md handoff]
```

| Step | What happens |
| --- | --- |
| **Locate** | Find the actual element, section, or file you meant |
| **Read** | Inspect tokens, conventions, and neighbors before proposing anything |
| **Gauge** | Trivial + specific? Just do it. Open-ended? Open the option space |
| **Propose** | Distinct directions with previews вҖ” one bolder than asked, exits always available |
| **Converge** | Expand once, then commit. No endless interrogation |
| **Build** | Implement in your house style; lean on expert skills where installed |
| **Verify** | Check the result against the *specific* element you pointed at |
| **Keep or switch** | Re-offer the directions you didn't pick вҖ” choices stay reversible |
| **Handoff** | On wrap-up, update `PROGRESS.md` so the next session resumes cold |

## What makes it different

These guardrails keep it useful instead of annoying:

| | Principle | What it means |
| ---: | --- | --- |
| рҹ”ҷ | **Choices stay reversible** | After it builds, it re-offers the other directions. Picking one never buries the rest. |
| рҹҡӘ | **Always an off-ramp** | *"I'll take it from here"* and *"Let's talk it through"* are always reachable вҖ” but real options come first. |
| рҹҸ  | **Borrow ideas, build in house style** | Research external patterns for structure; execute 100% in your tokens, type, and components. Nothing looks transplanted. |
| рҹ§  | **Stands on expert skills** | Animation вҶ’ official GSAP skills. Typography вҶ’ [FontPlaybook](https://github.com/MilkDeLeche/FontPlaybook). Says which one it's using. |
| рҹ§ӯ | **Leaves a breadcrumb** | Wrap-up updates `PROGRESS.md` in place вҖ” goal, done, what's left, how to resume. |
